作业 1 · 实战 Skills

用金融行业 Skills 完成一次真实调研

安装并使用 Anthropic 提供的 financial-services 这套 Skills， 先让 AI 帮你把仓库里所有 Skill 介绍汇总一遍，再挑一个真实任务， 组合多个 Skill 跑出一份完整产物。

  • 1
    安装 Skills
    克隆 / 安装 anthropics/financial-services 仓库下的 Skills 到 Claude Code。
  • 2
    汇总介绍
    让 AI 通读所有 Skill，整理成一份可读的索引（每个 Skill 是什么、能做什么、何时用）。
  • 3
    选定任务
    调研一家公司 / 调研一个行业 / 做一次财务分析 —— 任选其一。
  • 4
    编排工作流
    至少串联 2–3 个 Skill，按顺序跑完，产出一份完整调研报告。
  • 5
    提交到 GitHub
    把过程、产物、思考一起提交，方便复盘和复用。
参考样例： crusoe_questions_summary.md 展示了一条「认识工具 → 选用 → 执行 → 升级形态 → 沉淀方法」的完整链路 —— 你的作业不必照搬，但可以照着结构走。

Workflow · 作业 1 推荐流程

从认识 Skill 到产出报告

不要一上来就让 AI「调研一家公司」。先摸清工具，再编排流程，再执行。

1

装 + 看

安装 financial-services 下的 Skills，让 AI 读完仓库，输出一份 Skill 索引。

2

选任务

挑一个具体目标：例如调研 crusoe.ai、分析某个赛道、做财报快读。

3

编排顺序

让 AI 推荐用哪几个 Skill、按什么顺序，确认后按步骤跑。

4

沉淀

报告 + 提示词清单 + GitHub 提交，下次直接复用这条工作流。

Tips

几个小建议

🧭 先别急着「干活」

第一步永远是先让 AI 把工具摸清楚 —— Skill 的介绍、输入输出、适用场景。这一步省下的时间比你想的多。

🔗 工作流贵在「串」

单个 Skill 是工具，多个 Skill 串起来才是工作流。试着让上一步的输出，自然成为下一步的输入。

📝 把提示词留下来

每次 AI 跑完，把你「问了什么、为什么这么问」记一份 questions_summary.md，下次复用就是几秒钟的事。

🎯 spec 不必完美

作业 2 的目标是「定方向」，不是「定终稿」。先写出一个能讨论的版本，后面会随着课程一起迭代。

Why it matters

从「会用 AI」到「会编排 AI」

作业 1 让你第一次真正用别人写好的 Skills 解决一个真实任务 —— 这是未来你和 AI 协作的常态。 作业 2 把视线拉到自己身上：你想做的那个产品，到底是什么样？ 把它写下来，课程剩下的几次课就有了主线。
